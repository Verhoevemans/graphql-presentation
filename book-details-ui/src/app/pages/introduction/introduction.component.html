GraphQL versus REST

GraphQL Architecture

GraphQL is a query language for API's





Provides a complete and understandible description of the data in your API


Gives clients the power to ask for exactly what they need, and nothing more


Makes it easier to use and evolve API's over time





"GraphQL enables declarative data fetching where a client can specify exactly what it needs from an API, and the server only exposes a single endpoint"

Differences between GraphQL and REST





A REST setup often tries to offer an API that fits the requirements of all subscribing clients (overfetching and underfetching)
